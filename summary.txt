                            1. PROJECT OVERVIEW
================================================================================

Azure Haven is a full-stack hotel booking website that allows users to browse
rooms, make bookings, and manage their reservations. Administrators can manage
rooms, approve/reject bookings, and manage user accounts.

The application consists of:
- Frontend: Angular-based single-page application (SPA)
- Backend: RESTful API built with Express.js
- Database: MongoDB for data persistence

================================================================================
                        3. AUTHENTICATION SYSTEM
================================================================================

A. REGISTRATION:
   - Users can register with the following fields:
     * Username (unique, required)
     * Password (minimum 6 characters, required)
     * Email (unique, required)
     * Phone number (required)
     * City (required)
   - Default role is "user"
   - Password is hashed using bcryptjs (10 salt rounds)

B. LOGIN:
   - Users login with username and password
   - JWT token generated upon successful login
   - Token stored in localStorage
   - Token expires in 1 day
   - Redirects based on user role:
     * Admin → /admin dashboard
     * User → /user dashboard

C. TOKEN-BASED AUTHENTICATION:
   - JWT tokens used for secure API access
   - Auth interceptor automatically adds token to requests
   - Protected routes require valid token
   - Token contains user ID and role

================================================================================
                          7. BOOKING SYSTEM
================================================================================

BOOKING PROPERTIES:
- userId: Reference to User
- roomId: Reference to Room
- checkIn: Check-in date
- checkOut: Check-out date
- guests: Number of guests
- totalPrice: Calculated total (nights × price)
- status: Booking status (pending, confirmed, rejected)
- specialRequests: Optional user notes
- createdAt: Timestamp
- updatedAt: Timestamp

BOOKING WORKFLOW:
1. User selects a room
2. User fills booking form (dates, guests, special requests)
3. System validates availability and inputs
4. Booking created with "pending" status
5. Room status automatically changes to "occupied"
6. Admin reviews and approves/rejects booking
7. If approved: booking status = "confirmed"
8. If rejected: booking status = "rejected"
9. Room status syncs with booking status

PRICE CALCULATION:
- Total Price = Number of Nights × Room Price per Night
- Number of Nights = Check-out Date - Check-in Date

                          8. API ENDPOINTS
================================================================================

AUTH ROUTES (/api/auth):
- POST /register     - Register new user
- POST /login        - User login
- GET /users         - Get all users (admin only)

USER ROUTES (/api/users):
- GET /              - Get all users (admin only)
- GET /:id           - Get user by ID
- PATCH /:id         - Update user
- DELETE /:id        - Delete user

ROOM ROUTES (/api/rooms):
- POST /             - Create new room
- GET /              - Get all rooms
- GET /:id           - Get room by ID
- PATCH /:id         - Update room
- DELETE /:id        - Delete room

BOOKING ROUTES (/api/bookings):
- POST /             - Create new booking
- GET /              - Get all bookings
- GET /:id           - Get booking by ID
- PATCH /:id/status  - Update booking status
- DELETE /:id        - Delete booking
- GET /stats/:userId - Get user booking statistics
